# WP0/WP1 implementation work log

## Design packet

Goal:

Bootstrap the reviewable Rust repository controls required by WP0, then implement the frozen-draft WP1 primitive types, SHA-256 framing, `ResourceWireV1` codec, and stable decoder rejects.

Change class:

Class C, semantic implementation of an existing draft codec and identifier derivation. No canonical schema change is intended.

Affected crates and modules:

- workspace and repository policy files;
- `zrm-types` opaque primitives and reject codes;
- `zrm-crypto` reference SHA-256 framing;
- `zrm-codec` strict `ResourceWireV1` decoding, encoding, and `ResourceId` derivation;
- conformance, vector, fuzz, and Kani support.

Exact typed APIs:

```text
[u8; 32] -> TryFrom -> opaque nonzero identifier/root newtype
&[u8] -> decode_resource_wire_v1 -> ResourceWireV1 | ResourceWireDecodeError
ResourceWireV1 -> encode -> canonical bytes
ResourceWireV1 -> resource_id -> ResourceId
u32 -> TryFrom -> RejectCodeV1
```

Authority boundary:

WP1 constructs inert syntactic wire values and validated primitive values. It does not construct `ResourceV1`, verified facts, policies, state, commit plans, journals, or any other authority capability. A `ResourceId` remains data until a later semantic stage recomputes and validates the resource.

Invariants preserved or added:

- opaque 32-byte semantic types reject all-zero values;
- the reference suite ID uses the exact specified SHA-256 preimage;
- `ResourceWireV1` accepts only the 18 tagged fields in exact order and length;
- byte bounds are checked before parsing;
- truncation, tag, length, option, and trailing rejects follow the specified precedence;
- accepted bytes re-encode identically;
- `ResourceId` includes both required length frames and the `zrm.resource.v1` domain;
- parsing is deterministic, side-effect free, bounded, and panic-free for arbitrary byte slices.

Disaster states and CBC obligations:

- ambiguous or malleable resource bytes: `ZRM-CBC-001`;
- zero or cross-domain identifiers: `ZRM-CBC-002`;
- overflow or lossy conversion: `ZRM-CBC-016`;
- unbounded or differential decoding: `ZRM-CBC-019`;
- unstable reject precedence: `ZRM-CBC-024`;
- silent schema reinterpretation: `ZRM-CBC-026`;
- dependency-direction contamination: `ZRM-CBC-031`;
- unsafe core code: `ZRM-CBC-032`;
- unauditable complexity: `ZRM-CBC-033`;
- parser fuzz coverage: `ZRM-CBC-035`;
- agent contribution without evidence: `ZRM-CBC-039`;
- ungoverned dependencies/build tooling: `ZRM-CBC-040`;
- resource exhaustion: `ZRM-CBC-045`.

Canonical bytes or hashes affected:

Implementation only. The code must reproduce specification version `0.1.0-draft.2`: 595-byte absent-expiry wire values, 603-byte present-expiry wire values, the fixed suite ID, and the exact inner/outer length-framed `ResourceId` preimage.

Compatibility and versioning impact:

No compatibility promise or stable package-wide ABI is added. Any divergence discovered between code and the frozen draft blocks implementation and returns to specification review.

Tests to add first or with code:

- accepted absent/present expiry fixtures;
- exact byte, preimage, suite-ID, and `ResourceId` vectors;
- independent Python vector replay;
- all-zero primitive rejects and distinct-type compile-time APIs;
- all twelve decoder outcomes and precedence collisions;
- deterministic structure-preserving mutation atlas promoted to regressions;
- round-trip and one-field mutation properties;
- arbitrary-input fuzz target;
- bounded Kani constructor, cursor-totality, and ingress-precedence harnesses.

Formal and model obligations:

Kani is scoped to one opaque-constructor predicate, cursor totality for bounded symbolic lengths, and short-header ingress precedence. A full symbolic decode/encode round-trip harness exceeded the local resource budget and was removed after its timeout; it remains an explicit gap. Fuzzing and boundary-concolic exploration are bug-discovery evidence, not correctness proofs. Full codec functional proof and all semantic proofs remain later work-package obligations.

Dependency impact:

- `sha2` is the sole runtime dependency, exact-pinned with default features disabled;
- `libfuzzer-sys` is exact-pinned in the isolated fuzz workspace and is not part of the semantic runtime;
- repository scripts use the Python standard library only.

Performance and resource bounds:

- decoder work is linear in at most `MAX_RESOURCE_BYTES = 16 KiB`;
- decoder performs no allocation proportional to declared field lengths;
- encoder allocates exactly 595 or 603 bytes;
- hash preimage allocation is bounded by the canonical wire size.

Non-claims and known gaps:

- no validated `ResourceV1`;
- no state machine, persistence, exact-once runtime, proof adapter, or application adapter;
- no production, audit, privacy, availability, or reproducibility claim;
- branch protection, private vulnerability reporting, and release-environment approval require repository-host configuration and cannot be established by local files alone;
- human semantic and authority-boundary review remains required.
