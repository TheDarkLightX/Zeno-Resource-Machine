# Requests for comments

Breaking protocol, authority, or release changes require an approved RFC based on the repository template.

## Drafts

- [`RFC-0001: Policy, Context, Fact Freshness, and Commit Authority`](RFC-0001-policy-context-fact-and-commit-authority.md) — Class E; defines policy disposition, context-bound facts and plans, complete commit effects, atomic commit, crash recovery, idempotent retry, and admission/postcommit separation.

No RFC is approved until the maintainer records a decision after the required independent review.
